# Job Interview Preparation Markdown Workflow

Use this reference when generating a job-specific interview-prep Markdown document based on job information and project code paths.

## Required Structure

Include a table of contents and these sections:

1. 岗位关键词拆解
2. 项目与岗位匹配度总结
3. 面试开场总述
4. One section per relevant project
5. 面试表达主线

For each project section, include:

- 匹配点
- 1 分钟版
- 3 分钟版
- 追问版

## Project Ranking

Rank projects by relevance to the job. Lead with the strongest project even if it is not the newest.

For each project, connect it to the job using:

- Business scenario match.
- Technical stack match.
- Performance/scale match.
- Platform/engineering match.
- Seniority/owner/architecture match.

## 1 分钟版

Purpose: quick answer to “介绍一下这个项目”.

Include:

- Project business context.
- Your role.
- Core technical stack.
- Main challenges.
- Why it matches the target job.

Keep it short enough to speak naturally.

## 3 分钟版

Purpose: expanded project narration.

Include:

- Background and goal.
- Architecture or key modules.
- Your main responsibilities.
- Technical challenges and solutions.
- Result, scale, or transferable value.

## 追问版

Do not list questions only. Use the format:

```markdown
#### 追问 1：问题？

可以答：

回答内容。
```

Good 追问 topics:

- Why this architecture/design?
- How performance was optimized?
- How compatibility was handled?
- How state/data flow worked?
- What went wrong or could be improved?
- How this experience transfers to the target job?

## Style

Use interview-ready Chinese. Be direct and practical. Make answers defensible from code and resume evidence.
